# Attribute Group Model

**Project:** Ocypus Nexus

**Document Version:** 1.0

**Status:** Released

---

# 1. Purpose

This document defines the standard model for Attribute Groups in Ocypus Nexus.

Attribute Groups organize related Attributes into logical sections.

Groups are used consistently across:

- Product Edit Pages
- Product Detail Pages
- Excel Templates
- REST APIs
- Product Comparison
- Product Search
- Documentation

The same group definition must be reused throughout the platform.

---

# 2. Design Principles

Attribute Groups follow these principles:

- Metadata Driven
- Reusable
- Category Independent
- Ordered
- Version Controlled
- Configurable
- UI Independent

Groups contain metadata only.

Groups never store product values.

# 6. Group Code Rules

Group Codes use **UPPER_SNAKE_CASE**.

Examples

```
GENERAL

COOLING

COMPATIBILITY

PACKAGE

MARKETING
```

Rules

- English only
- Uppercase
- Underscore separator
- Globally unique
- Immutable after release

# 8. Category Scope

Groups may be:

## Global

Available to all categories.

Examples:

- General
- Package
- Downloads

## Category Specific

Available only to selected categories.

Examples:

Cooling Group

Applies to:

- Case
- Liquid Cooler
- Air Cooler

Storage Group

Applies to:

- Case

# 9. UI Rendering

Each Group renders as one section.

Examples

Product Edit Page

```
General Information
-------------------------

Dimensions
-------------------------

Compatibility
-------------------------

Cooling
-------------------------
```

# 11. API Mapping

REST APIs return grouped metadata.

Example

```json
{
    "general": { },
    "dimension": { },
    "compatibility": { },
    "cooling": { }
}
```

Groups improve readability but do not change the underlying data model.

---

# 15. Version Control

Each Group stores:

- Version
- Status
- Created Time
- Updated Time
- Created By
- Updated By

Historical versions remain queryable.

---

# 16. Audit Requirements

Every modification creates an audit record.

Audit includes:

- User
- Timestamp
- Action
- Previous Value
- New Value
- Change Reason

Audit records cannot be deleted.

---

# 17. Business Rules

- Group Codes are globally unique.
- Groups may contain unlimited Attributes.
- An Attribute belongs to exactly one Group.
- Empty Groups are allowed during configuration.
- Hidden Groups are excluded from exports unless explicitly enabled.
- Groups are soft deleted only.

---

# 18. Definition of Done

The Attribute Group Model is complete when:

- Groups are metadata-driven.
- UI sections are generated from Groups.
- Excel sections are generated from Groups.
- APIs support Group-based organization.
- Search panels are Group-based.
- Localization is supported.
- Version history is maintained.
- Audit logs are generated automatically.
